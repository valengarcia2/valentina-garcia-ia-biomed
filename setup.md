## 1. ¿Qué vamos a construir en esta materia?

Esta materia no busca simplemente enseñar herramientas. Buscamos que desarrolles una forma **profesional de trabajar con IA en problemas biomédicos**.

Durante la cursada cada estudiante construirá:

- Su propio repositorio profesional.
- Un historial versionado de su trabajo.
- Experimentos documentados.
- Implementaciones en Python.
- Uso criterioso de modelos generativos como herramienta.

---

## 2. Conceptos Fundamentales

| Término | Definición |
|---------|------------|
| **IDE** | Tu laboratorio digital. Editor desde donde escribís código, organizás proyectos y ejecutás experimentos. |
| **Git** | Sistema que registra los cambios y versiones en tu proyecto. |
| **GitHub** | Git guardado en internet. |
| **Repositorio** | Una carpeta donde Git trackea las versiones (1 por proyecto de desarrollo). |
| **Librería** | Programas ya escritos en un lenguaje para llevar adelante tareas específicas. |
| **Entorno** | Las librerías y herramientas de desarrollo instaladas necesarias para que tu proyecto corra. |

## 12. Cómo empezar: Clonar el repositorio base

### En GitHub

1. Abrir [https://github.com/mszsorondo/ia-biomed](https://github.com/mszsorondo/ia-biomed)
2. Click en **"Use this template"** → *Create a new repository*.
3. Asignarle un nombre y presionar **"Create Repository"**.
4. Una vez creado, copiar el link al repositorio.

### En Antigravity

1. Abrir Antigravity.
2. Click en **"Clone repository"**.
3. Elegir una carpeta donde clonar el repositorio.
4. Click en **"Open"** — Antigravity abrirá el repositorio como proyecto.

Ahora tenés una copia local del repositorio que podés modificar y pushear a GitHub.

### Ejemplo: Agregar cambios al repositorio de GitHub

1. Crear la carpeta `Entrega 0`.
2. Crear un archivo `entrega.txt` en esa misma carpeta, escribir una frase cualquiera y guardar.
3. Click en **"Source Control"** (símbolo de rama en el panel izquierdo).
4. Click en **Commit**.
5. Escribir el mensaje del commit.
6. Click en **"Synch Changes"**.
7. Loguearse a GitHub.
